# 4.5 代码推送与部署

## 目标

在本节中，你将：
- 学习如何将代码推送到cnb.cool仓库
- 掌握如何在cnb.cool上部署静态网站
- 学会验证网站是否成功部署

---

## 步骤1：提交本地代码

在推送代码之前，我们需要先将代码提交到本地Git仓库。

### 操作步骤：

1. **查看当前状态**
   ```bash
   git status
   ```
   
   查看哪些文件被修改了

2. **添加文件到暂存区**
   ```bash
   git add .
   ```
   
   或者只添加特定文件：
   ```bash
   git add index.html
   git add css/style.css
   git add js/script.js
   ```

3. **提交代码**
   ```bash
   git commit -m "初始化项目：添加基础网页文件"
   ```
   
   提交信息应该简洁明了，描述这次提交做了什么

4. **查看提交历史**
   ```bash
   git log --oneline
   ```
   
   确认提交成功

---

## 步骤2：推送代码到cnb.cool

现在让我们将代码推送到cnb.cool的远程仓库。

### 操作步骤：

1. **确认远程仓库配置**
   ```bash
   git remote -v
   ```
   
   确认cnb远程仓库已正确配置

2. **推送代码**
   ```bash
   git push cnb main
   ```
   
   如果是第一次推送，可能需要使用：
   ```bash
   git push -u cnb main
   ```

3. **验证推送成功**
   - 打开浏览器，访问你的cnb.cool仓库页面
   - 确认代码文件已经出现在仓库中
   - 查看提交历史，确认你的提交已经记录

---

## 步骤3：配置网站部署

现在让我们配置cnb.cool的网站部署功能。

### 操作步骤：

1. **进入项目设置**
   - 在cnb.cool仓库页面，点击「Settings」或「设置」
   - 找到「Pages」或「网站部署」相关选项

2. **配置构建设置**
   - **构建目录**：输入 `project`（因为我们的网页文件在project目录中）
   - **构建命令**：留空（静态网站不需要构建）
   - **输出目录**：与构建目录相同，或者留空

3. **启用自动部署**
   - 勾选「自动部署」或「Auto Deploy」选项
   - 这样每次推送代码时，网站会自动更新

4. **保存配置**
   - 点击「保存」或「Save」按钮
   - 等待配置生效

---

## 步骤4：部署网站

现在让我们手动触发一次部署，或者等待自动部署完成。

### 操作步骤：

1. **手动触发部署**
   - 如果没有启用自动部署，点击「部署」或「Deploy」按钮
   - 等待部署过程完成

2. **查看部署状态**
   - 在仓库页面查看部署状态
   - 通常会显示「部署中」、「部署成功」或「部署失败」

3. **获取网站URL**
   - 部署成功后，会显示网站的访问URL
   - 格式通常为：`https://cnb.cool/<你的用户名>/<仓库名>`
   - 或者：`https://<仓库名>-<你的用户名>.cnb.app`

---

## 步骤5：访问和验证网站

现在让我们访问部署好的网站，验证一切正常。

### 操作步骤：

1. **打开网站**
   - 在浏览器中输入部署成功后的URL
   - 或者点击cnb.cool提供的「Site」或「访问网站」按钮

2. **验证页面内容**
   - 确认页面正常显示
   - 检查标题、文本内容是否正确

3. **验证样式和脚本**
   - 确认CSS样式正常应用
   - 测试JavaScript交互功能
   - 检查图片和其他资源是否正常加载

4. **测试响应式设计**
   - 如果有响应式设计，测试不同屏幕尺寸
   - 使用浏览器的开发者工具检查

## 部署最佳实践

### 1. 代码管理
- 📝 **有意义的提交信息**：清晰描述每次提交做了什么
- 🏷️ **使用标签**：为重要版本打标签
- 🌿 **分支管理**：使用分支开发新功能

### 2. 部署管理
- ⏰ **自动部署**：启用自动部署，提高效率
- 📋 **部署检查清单**：部署前检查关键内容
- 📊 **监控访问**：定期查看网站访问情况

### 3. 安全注意事项
- 🔒 **保护访问令牌**：不要将令牌提交到代码仓库
- 🧹 **清理敏感信息**：确保代码中不包含敏感数据
- 🔄 **定期更新**：保持依赖和工具更新

## 常见问题与解答

### Q: 推送代码时提示认证失败怎么办？
**A:** 检查访问令牌是否正确，确认令牌没有过期，必要时创建新令牌。

### Q: 部署失败怎么办？
**A:** 检查构建配置是否正确，特别是构建目录是否指向正确的目录。查看部署日志，了解具体的错误信息。

### Q: 如何回滚到之前的版本？
**A:** 在cnb.cool的部署历史中，找到之前的成功部署，点击「回滚」或「Rollback」按钮。

### Q: 可以绑定自定义域名吗？
**A:** 可以，在cnb.cool的项目设置中，添加自定义域名，然后在域名提供商处配置DNS记录。

### Q: 网站访问速度慢怎么办？
**A:** cnb.cool提供CDN加速，通常访问速度很快。如果遇到问题，可以检查网络连接或联系客服。
