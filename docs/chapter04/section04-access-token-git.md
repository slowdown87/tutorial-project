## 什么是访问令牌？

**访问令牌（Access Token）**是一个用于身份验证的字符串，它相当于你的密码，但更加安全和灵活。

### 为什么需要访问令牌？
- 🔐 **安全性**：比直接使用密码更安全
- 🎛️ **权限控制**：可以设置不同的权限级别
- 📅 **有效期**：可以设置过期时间
- 🗑️ **可撤销**：可以随时撤销，不影响主密码

### cnb.cool的认证方式：
- **用户名**：固定填写 `cnb`
- **密码**：填写你创建的访问令牌

---

## 步骤1：创建访问令牌

### 操作步骤：

1. **进入个人设置**
   - 登录cnb.cool后，点击右上角的用户头像
   - 选择「设置」或「Settings」
   - 或者点击「个人设置」

2. **找到访问令牌设置**
   - 在设置页面中，找到「访问令牌」或「Access Token」选项
   - 点击进入访问令牌管理页面

3. **创建新令牌**
   - 点击「创建新令牌」或「Generate New Token」按钮
   - 输入令牌名称（例如：`tutorial-project-token`）
     - 建议使用有意义的名称，方便识别
   - 选择令牌权限
     - **读写权限**：可以推送和拉取代码（推荐）
     - **只读权限**：只能拉取代码，不能推送
   - 可选：设置过期时间（增加安全性）
   - 点击「创建」或「Generate」按钮

4. **复制令牌**
   - ⚠️ **重要**：令牌只会显示一次！
   - 立即复制生成的令牌
   - 保存到安全的地方（例如密码管理器）
   - 不要分享给他人

## 步骤2：配置Git远程仓库

### 操作步骤：

1. **打开终端**
   - 在Solo Trae Web Code中打开终端
   - 或者在本地电脑打开命令行工具

2. **进入项目目录**
   ```bash
   cd /path/to/your/tutorial-project
   ```
   将 `/path/to/your/tutorial-project` 替换为实际的项目路径

3. **添加远程仓库**
   ```bash
   git remote add cnb https://cnb.cool/<你的用户名>/tutorial-project.git
   ```
   将 `<你的用户名>` 替换为你在cnb.cool上的用户名

4. **配置认证信息**
   
   方法1：在URL中包含认证信息（推荐）
   ```bash
   git remote set-url cnb https://cnb:<你的访问令牌>@cnb.cool/<你的用户名>/tutorial-project.git
   ```
   
   注意：
   - 将 `<你的访问令牌>` 替换为你刚才创建的令牌
   - 将 `<你的用户名>` 替换为你在cnb.cool上的用户名
   - 用户名固定为 `cnb`

   方法2：使用Git凭据存储
   ```bash
   git config credential.helper store
   ```
   然后在第一次推送时输入用户名和令牌

---

## 步骤3：验证Git配置

### 操作步骤：

1. **查看远程仓库配置**
   ```bash
   git remote -v
   ```
   
   你应该看到类似这样的输出：
   ```
   cnb  https://cnb:<你的访问令牌>@cnb.cool/<你的用户名>/tutorial-project.git (fetch)
   cnb  https://cnb:<你的访问令牌>@cnb.cool/<你的用户名>/tutorial-project.git (push)
   ```

2. **检查Git配置**
   ```bash
   git config --list
   ```
   
   确认用户名和邮箱配置正确

3. **测试连接**
   ```bash
   git ls-remote cnb
   ```
   
   如果配置正确，会显示远程仓库的引用信息

## 访问令牌安全最佳实践

### 1. 令牌管理
- 🔑 **不要分享**：不要将令牌分享给他人
- 💾 **安全存储**：使用密码管理器存储令牌
- 📅 **定期轮换**：定期创建新令牌，撤销旧令牌
- 🎯 **最小权限**：只授予必要的权限

### 2. 权限设置
- **读写权限**：用于需要推送代码的场景
- **只读权限**：用于只需要拉取代码的场景
- **特定仓库**：如果支持，尽量限制令牌的访问范围

### 3. 应急处理
- **令牌泄露**：如果令牌泄露，立即撤销
- **可疑活动**：发现异常活动时，立即检查并更换令牌
- **审计日志**：定期查看令牌使用日志

## 常见问题与解答

### Q: 访问令牌忘记保存怎么办？
**A:** 访问令牌只显示一次，如果忘记保存，需要撤销旧令牌并创建新令牌。

### Q: 可以创建多个访问令牌吗？
**A:** 可以，建议为不同的项目或用途创建不同的令牌，便于管理和撤销。

### Q: 访问令牌有过期时间吗？
**A:** 可以在创建时设置过期时间，也可以创建永不过期的令牌（不推荐）。

### Q: cnb.cool支持SSH吗？
**A:** cnb.cool目前不支持SSH协议，只支持HTTPS协议配合访问令牌进行认证。

### Q: 访问令牌泄露了怎么办？
**A:** 如果访问令牌泄露了，请立即在个人设置页面撤销该令牌，并创建一个新的令牌。
